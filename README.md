# Satellite Imagery Gap Filling
The goal of this project is to perform gap-filling for satellite imagery. Specifically, we look to fill the gaps of images generated by the [Geostationary Operational Environmental Satellite (GOES) -- R Series](https://www.goes-r.gov/), the nation's most advanced fleet of geostationary weather satellites. The product we are gap filling is [Aerosol Optical Depth (AOD)](https://www.earthdata.nasa.gov/topics/atmosphere/aerosol-optical-depth-thickness), which measures the aerosols distributed within a column of air from the Earth's surface to the top of the atmosphere. [^1]

Since this product relies on measuring the sunlight that bounce off of aerosols, events like heavy smoke or cloud cover can obscur large portions of the image.

![Aerosol Optical Depth](https://www.goes-r.gov/imagesContent/users/products/Aerosol-Optical-Depth.jpg)

# Prior work
Our work will be primarily based on the work of Lee et al. [^2] We'll be tackling the same problem, but swap out their data with different sources. The goal of our contribution will be to:
1. Use an AOD product with a higher spatial and temporal resolution.
2. Find and use independent variables that are able to match this new resolution.

Due to the increased spatial resolution, our current target area will be reduced from the entire continental US to just California.

## 1. Train (Docker)

Build the image once:

```bash
docker build -t aod-gapfill .
```

Run training, mounting the cache and an outputs directory.

**Template** (fill in the placeholders):

```bash
docker run --rm --gpus all \
  -v "$(pwd)/<CACHE_DIR>:/app/<CACHE_DIR>" \
  -v "$(pwd)/<OUTPUTS_DIR>:/app/<OUTPUTS_DIR>" \
  aod-gapfill --cache-dir /app/<CACHE_DIR> --epochs <N_EPOCHS>
```

**Example**:

```bash
docker run --rm --gpus all \
  -v "$(pwd)/cache_96:/app/cache_96" \
  -v "$(pwd)/outputs:/app/outputs" \
  aod-gapfill --cache-dir /app/cache_96 --epochs 20
```

Each run lands in a timestamped subdirectory under `outputs/` containing:

- `training_curve.png` — train + val loss per epoch
- `metrics.json` — pixel-wise and AERONET RMSE / r / NMB
- `aeronet_scatter.png` — predicted vs AERONET 550 nm
- `panel_all_timesteps.png` — obs / pred / diff for every valid hour, train/val labeled
- `prediction.png`, `prediction.npy`, `target_observed.npy` — single-hour comparison
- `snapshot.pt` — model weights + normalization stats
